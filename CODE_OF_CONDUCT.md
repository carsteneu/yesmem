# Code of Conduct — YesMem

## 1. Purpose

YesMem aims to provide a focused, high-signal environment for building and improving a local-first memory system for developer workflows.

We prioritize:

* technical clarity
* constructive collaboration
* long-term maintainability of the codebase

This Code of Conduct defines the expected behavior for all participants.

---

## 2. Scope

This Code of Conduct applies to all project spaces, including:

* GitHub repositories (issues, pull requests, discussions)
* documentation and code comments
* community interactions related to the project

It applies to:

* maintainers
* contributors
* users
* anyone interacting with the project

---

## 3. Expected Behavior

Participants are expected to:

* communicate clearly and directly
* provide constructive, actionable feedback
* focus discussions on technical substance
* respect differing approaches and levels of experience
* assume good intent unless clearly disproven

Examples:

* explaining why a change is needed, not just rejecting it
* proposing alternatives instead of vague criticism
* keeping discussions scoped and relevant

---

## 4. Unacceptable Behavior

The following is not tolerated:

* personal attacks or insults
* dismissive or hostile communication
* harassment, discrimination, or exclusionary behavior
* low-effort noise (spam, trolling, repeated derailment)
* sharing private information without consent

---

## 5. Decision Authority

Maintainers have final authority over:

* codebase direction
* architectural decisions
* acceptance or rejection of contributions

Decisions are expected to be:

* technically justified
* consistent
* documented where relevant

---

## 6. Enforcement

Maintainers may take action in response to violations, including:

* requesting changes in behavior
* limiting interaction (e.g., locking threads)
* removing contributions
* temporary or permanent bans

Severity and context will be considered.

---

## 7. Reporting

If you encounter behavior that violates this Code of Conduct, report it via:

[Insert contact method — e.g. GitHub issue, email, or private channel]

Reports will be handled:

* confidentially where possible
* promptly
* with a focus on fairness

---

## 8. Maintainer Responsibilities

Maintainers are expected to:

* enforce this Code of Conduct consistently
* avoid arbitrary or opaque decisions
* maintain a high signal-to-noise ratio in discussions
* model the behavior expected from contributors

---

## 9. Project-Specific Principles

Given the nature of YesMem:

* discussions should favor technical depth over verbosity
* claims should be backed by implementation details or measurable effects
* design debates should converge — avoid endless philosophical loops without actionable outcomes

---

## 10. Attribution

This Code of Conduct is inspired by established open-source practices and adapted to the specific goals of this project.
